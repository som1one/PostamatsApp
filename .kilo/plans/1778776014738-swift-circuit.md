# Plan: Запуск фронта + бека, кнопки возврата/переноса, мобильная адаптация

## Контекст проекта

- **Backend**: FastAPI (Python), порт 8000, SQLite в dev, PostgreSQL в prod
- **Frontend**: Next.js 16 (React 19), порт 3001
- **API base**: `NEXT_PUBLIC_API_BASE_URL=http://127.0.0.1:8010` — фронт смотрит на порт 8010, но бек запускается на 8000. Нужно исправить.
- **CORS**: бек разрешает `http://127.0.0.1:3001` в DEBUG режиме
- **Оплата**: dev-stub включён (`YOOKASSA_DEV_STUB=true`, `NEXT_PUBLIC_ENABLE_DEV_PAYMENT_BYPASS=true`)

## Задача 1: Запуск фронта и бека (фронт тянет данные из бека)

### Проблема
`web/.env.local` указывает `NEXT_PUBLIC_API_BASE_URL=http://127.0.0.1:8010`, но бек слушает порт **8000**. Нужно исправить на `8000`.

### Действия
1. Изменить `web/.env.local`: `NEXT_PUBLIC_API_BASE_URL=http://127.0.0.1:8000`
2. Изменить `backend/.env`: переключить DB на SQLite (PostgreSQL не запущен локально, `dev.sqlite3` уже существует)
   - `DB_URL=sqlite+aiosqlite:///./backend/dev.sqlite3`
   - `ASYNC_DB_URL=sqlite+aiosqlite:///./backend/dev.sqlite3`
   - Убрать/закомментировать PostgreSQL-специфичные строки
3. Убедиться что CORS в `backend/.env` включает `http://127.0.0.1:3001` (уже есть через DEBUG=true)
4. Запустить бек: `python -m uvicorn backend.main:app --host 127.0.0.1 --port 8000 --reload` из корня проекта
5. Запустить фронт: `npm run dev` из `web/`

## Задача 2: Кнопка возврата денег (после оплаты) с диалогом

### Где находится кнопка отмены
Файл: `web/src/app/payment/return/PaymentReturnClient.tsx`

Текущая кнопка "Отменить оплату" (строки 264–273) — уже существует, но:
- Нет диалога подтверждения
- Нет предложения перенести запись

### Новое поведение кнопки "Отменить оплату"
При нажатии — показывать **модальный диалог** с текстом:
> «Вы уверены, что хотите вернуть деньги? Вы можете перенести запись на другое время.»

Кнопки в диалоге:
- **«Перенести запись»** → переход на `/catalog` (или страницу выбора даты, если она есть)
- **«Да, вернуть деньги»** → выполняет `cancelReservation` (текущая логика)
- **«Назад»** → закрывает диалог

### Реализация
- Добавить state `showCancelDialog: boolean` в `PaymentReturnContent`
- Кнопка "Отменить оплату" → устанавливает `showCancelDialog = true`
- Модальный диалог использует существующие CSS классы `.modal-overlay`, `.modal-box`, `.modal-title`, `.modal-text`, `.modal-actions`
- Кнопка "Перенести" → `router.push('/catalog')` + `clearPendingCheckout()`
- Кнопка "Вернуть деньги" → вызывает `handleCancelPayment()` (существующая логика)

## Задача 4: Мобильная адаптация

### Проверить и исправить
CSS уже содержит медиа-запросы для мобилки. Нужно убедиться что:

1. **Модальные диалоги** — `.modal-*` классы уже есть в globals.css (строки 7278–7336), но используют несуществующие CSS-переменные (`--border`, `--radius-lg`, `--surface-raised`, `--accent`, `--text-primary`, `--text-secondary`). Нужно исправить на переменные проекта.
2. **Кнопки в диалоге** (`.modal-actions`) — нет мобильного breakpoint. Добавить `flex-direction: column` на `≤480px`.
3. **Страница `/payment/return`** — `layout-split` уже переключается в одну колонку на `≤1080px` (есть в CSS).
