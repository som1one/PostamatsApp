# Bugfix Requirements Document

## Introduction

В клиентском веб-интерфейсе бронирования, которые уже были завершены пользователем (вещь возвращена / нажата кнопка «Завершить» / статус переведён в `completed`), после прохода планового времени окончания (`plannedEndAt` / `end_time`) снова отображаются с пометкой «Просрочено».

Исходный отчёт пользователя:

> «Завершённые бронирования отображаются как «Просрочено» после прохода времени окончания (end_time) в клиентском веб-интерфейсе.»

Шаги воспроизведения, предоставленные пользователем:

1. Создал бронирование с `end_time = 12:00`.
2. В 11:55 статус был «Активно / Завершается».
3. Пользователь вернул вещь / нажал «Завершить» в 11:58 — статус стал «Завершено».
4. В 12:01 (после прохода `end_time`) статус снова показывается как «Просрочено».

Дефект проявляется только в клиентском веб-интерфейсе (`web/`), а не в админ-панели. Бэкенд корректно сохраняет терминальный статус бронирования (`RentalStatus.COMPLETED`) и не возвращает его в `OVERDUE` — фоновый воркер `mark_overdue_rentals` переводит в `OVERDUE` только записи со статусом `ACTIVE`. Таким образом, пользователь видит ложный «просроченный» статус, что вводит в заблуждение и подрывает доверие к системе.

Завершённое бронирование — это терминальное состояние: после возврата вещи оно должно оставаться «Завершено» навсегда, независимо от того, прошло ли плановое время окончания. Помечаться как «Просрочено» должны только те бронирования, которые НЕ были завершены вовремя.

## Bug Analysis

### Current Behavior (Defect)

В клиентском веб-интерфейсе виджет «дедлайна» бронирования (deadline meta) определяет, что бронирование «просрочено», на основании сравнения текущего времени с `plannedEndAt`, не учитывая, что бронирование уже могло быть завершено.

1.1 WHEN бронирование имеет терминальный статус «завершено» (`completed`) И текущее время позднее `plannedEndAt` THEN клиентский веб-интерфейс отображает бронирование с плашкой «Просрочено на {длительность}» вместо «Завершено»

1.2 WHEN бронирование имеет терминальный статус «завершено» (`completed`) И текущее время позднее `plannedEndAt` THEN клиентский веб-интерфейс предлагает пользователю «оформить возврат как можно скорее», хотя возврат уже выполнен

### Expected Behavior (Correct)

Завершённое бронирование является терминальным состоянием: после установления статуса `completed` бронирование должно отображаться как «Завершено» постоянно, даже если плановое время окончания уже прошло.

2.1 WHEN бронирование имеет терминальный статус «завершено» (`completed`) И текущее время позднее `plannedEndAt` THEN клиентский веб-интерфейс SHALL отображать бронирование как «Завершено» (без плашки «Просрочено»)

2.2 WHEN бронирование имеет терминальный статус «завершено» (`completed`) И текущее время позднее `plannedEndAt` THEN клиентский веб-интерфейс SHALL NOT предлагать пользователю оформлять возврат

### Unchanged Behavior (Regression Prevention)

Поведение для незавершённых бронирований и для всех остальных сценариев должно остаться без изменений.

3.1 WHEN бронирование имеет статус `active` И текущее время позднее `plannedEndAt` THEN клиентский веб-интерфейс SHALL CONTINUE TO отображать бронирование как «Просрочено на {длительность}»

3.2 WHEN бронирование имеет статус `overdue` THEN клиентский веб-интерфейс SHALL CONTINUE TO отображать бронирование как «Просрочено на {длительность}»

3.3 WHEN бронирование имеет статус `active`, `pickup_ready` или `pickup_opened` И текущее время раньше `plannedEndAt` THEN клиентский веб-интерфейс SHALL CONTINUE TO отображать плашку «До возврата: {длительность}»

3.4 WHEN бронирование имеет статус `return_in_progress` THEN клиентский веб-интерфейс SHALL CONTINUE TO отображать плашку «Возврат уже начат»

3.5 WHEN бронирование имеет статус `cancelled` THEN клиентский веб-интерфейс SHALL CONTINUE TO отображать существующее представление отменённого бронирования (включая случай `cancelReason = pickup_expired`)

3.6 WHEN бронирование имеет терминальный статус «завершено» (`completed`) И текущее время раньше `plannedEndAt` THEN клиентский веб-интерфейс SHALL CONTINUE TO отображать бронирование как «Завершено»

### Bug Condition

```pascal
FUNCTION isBugCondition(rental, now)
  INPUT: rental of type RentalListItem (status, plannedEndAt, ...),
         now of type Timestamp
  OUTPUT: boolean

  // Бронирование уже находится в терминальном "завершённом" состоянии,
  // но плановое время окончания уже прошло, из-за чего UI ошибочно
  // помечает его как "Просрочено".
  RETURN rental.status = "completed"
         AND rental.plannedEndAt IS NOT NULL
         AND now > rental.plannedEndAt
END FUNCTION
```

### Property: Fix Checking

```pascal
// Property: Fix Checking — завершённое бронирование остаётся «Завершено»
FOR ALL (rental, now) WHERE isBugCondition(rental, now) DO
  meta ← getRentalDeadlineMeta'(rental, now)
  ASSERT meta.tone ≠ "danger"
  ASSERT meta.title NOT CONTAINS "Просрочено"
  // Дополнительно: интерфейс не должен предлагать оформить возврат
  ASSERT canOfferReturn'(rental) = false
END FOR
```

### Property: Preservation Checking

```pascal
// Property: Preservation Checking — для всех остальных входов поведение не меняется
FOR ALL (rental, now) WHERE NOT isBugCondition(rental, now) DO
  ASSERT getRentalDeadlineMeta(rental, now) = getRentalDeadlineMeta'(rental, now)
  ASSERT canOfferReturn(rental) = canOfferReturn'(rental)
END FOR
```
