# Bugfix Requirements Document

## Introduction

Карточки товаров (`article.card.product-card`) на сетке каталога (`.product-grid`)
выглядят обрезанными в развёрнутом окружении (на сервере / в production-сборке),
тогда как в локальной разработке они отображаются нормально. У части карточек
обрезается содержимое: название товара (`h3`), краткое описание
(`.product-description`), факты (`.product-facts`) или нижний блок с ценой и
кнопкой «Выбрать» (`.product-meta`). Причина — комбинация `overflow: hidden` на
`.product-card` с растягиванием карточки по высоте сетки (`height: 100%`,
`align-self: stretch`, `grid-auto-rows`/`align-items`), при которой контент
не помещается в выделенную высоту и срезается границей карточки. Эффект
проявляется на сервере из-за иного порядка/слияния CSS в собранном бандле
(несколько конфликтующих блоков `.product-card` в `globals-04.css` и
`globals-08.css`), из-за чего итоговая высота строки сетки и видимая область
карточки отличаются от локальной отрисовки.

Цель — сделать так, чтобы карточки товаров полностью показывали своё
содержимое и не были обрезаны в развёрнутом окружении, при этом сохранив
текущую раскладку и поведение там, где обрезки нет.

## Bug Analysis

### Current Behavior (Defect)

В развёрнутом окружении (production-сборка на сервере) содержимое карточек
товаров обрезается границей карточки.

1.1 WHEN карточка товара рендерится на сервере и её содержимое (название,
описание, факты, цена с кнопкой) суммарно выше высоты, выделенной строкой
сетки THEN система обрезает нижнюю часть карточки из-за `overflow: hidden`,
и часть содержимого становится невидимой

1.2 WHEN в одной строке `.product-grid` находятся карточки с разным объёмом
содержимого (например, разной длины название или наличие/отсутствие описания)
THEN система растягивает все карточки строки до высоты самой высокой
(`height: 100%` / `align-self: stretch`), но более «насыщенные» карточки всё
равно обрезаются, так как их контент превышает рассчитанную высоту строки

1.3 WHEN страница каталога собрана в production и применяется итоговый
объединённый CSS-бандл THEN система применяет конфликтующие правила
`.product-card` из `globals-04.css` и `globals-08.css` в порядке, отличном от
локального, из-за чего видимая высота карточки и положение нижнего блока
(`.product-meta` c `margin-top: auto`) отрисовываются иначе и содержимое
срезается

1.4 WHEN нижний блок `.product-meta` (цена и кнопка «Выбрать») выталкивается
вниз через `margin-top: auto`, а суммарная высота контента превышает высоту
карточки THEN система обрезает кнопку «Выбрать» и/или цену снизу

### Expected Behavior (Correct)

2.1 WHEN карточка товара рендерится на сервере и её содержимое выше выделенной
высоты THEN система SHALL показать всё содержимое карточки полностью без
обрезки (высота карточки SHALL подстраиваться под содержимое либо клампы/
ограничения высоты SHALL гарантировать, что контент помещается в видимую
область)

2.2 WHEN в одной строке `.product-grid` находятся карточки с разным объёмом
содержимого THEN система SHALL отображать каждую карточку с полностью видимым
содержимым (название, описание, факты, цена и кнопка «Выбрать»), сохраняя при
этом выровненную сетку

2.3 WHEN страница каталога собрана в production и применяется итоговый
CSS-бандл THEN система SHALL давать тот же визуальный результат, что и
локально: правила `.product-card` SHALL быть согласованы (без
взаимоисключающих значений высоты/overflow), чтобы порядок слияния CSS не
приводил к обрезке

2.4 WHEN нижний блок `.product-meta` (цена и кнопка «Выбрать») позиционируется
внизу карточки THEN система SHALL держать цену и кнопку «Выбрать» полностью
видимыми и кликабельными, не срезая их границей карточки

### Unchanged Behavior (Regression Prevention)

3.1 WHEN содержимое карточки помещается в выделенную высоту (обрезки нет) THEN
система SHALL CONTINUE TO отображать карточку с текущей раскладкой,
скруглением (`border-radius`), отступами и выравниванием сетки

3.2 WHEN применяется десктопная сетка `.product-grid`
(`repeat(3, minmax(0, 1fr))`) THEN система SHALL CONTINUE TO раскладывать
карточки в три колонки с текущим `gap`

3.3 WHEN применяются мобильные/планшетные брейкпоинты (двухколоночная сетка и
связанные переопределения в `globals-06/12/13/16/17.css`) THEN система SHALL
CONTINUE TO применять текущие адаптивные стили карточек (скрытие описания/
eyebrow, размеры шрифтов, паддинги)

3.4 WHEN название товара (`h3`) или описание (`.product-description`) длиннее
двух строк THEN система SHALL CONTINUE TO ограничивать их двумя строками через
`-webkit-line-clamp: 2` (кламп текста — это ожидаемое поведение, а не обрезка
карточки)

3.5 WHEN пользователь наводит курсор на карточку THEN система SHALL CONTINUE TO
показывать hover-эффект (тень/бордер/transform) без изменений

3.6 WHEN обложка товара (`.product-cover img`) отрисовывается THEN система
SHALL CONTINUE TO вписывать изображение через `object-fit: contain` в заданном
соотношении сторон без искажений

## Bug Condition and Property

### Bug Condition

```pascal
FUNCTION isBugCondition(X)
  INPUT: X of type ProductCardRender   // карточка с её содержимым и окружением рендера
  OUTPUT: boolean

  // Баг проявляется, когда суммарная высота содержимого карточки
  // превышает видимую высоту карточки в собранном (server/production) бандле,
  // и overflow: hidden срезает содержимое.
  RETURN X.environment = SERVER
         AND contentHeight(X) > visibleCardHeight(X)
         AND cardOverflow(X) = HIDDEN
END FUNCTION
```

### Property: Fix Checking

```pascal
// Свойство: всё содержимое карточки видимо, ничего не срезано
FOR ALL X WHERE isBugCondition(X) DO
  result ← renderProductCard'(X)
  ASSERT isFullyVisible(result.title)
     AND isFullyVisible(result.facts)
     AND isFullyVisible(result.price)
     AND isClickable(result.chooseButton)
     AND NOT isClipped(result.card)
END FOR
```

### Property: Preservation Checking

```pascal
// Свойство: для карточек без обрезки поведение не меняется
FOR ALL X WHERE NOT isBugCondition(X) DO
  ASSERT renderProductCard(X) = renderProductCard'(X)
END FOR
```

**Определения:**
- **F** (`renderProductCard`): отрисовка карточки до исправления.
- **F'** (`renderProductCard'`): отрисовка карточки после исправления.
- **Counterexample**: карточка с длинным названием в две строки + описанием +
  фактами + ценой и кнопкой, отрисованная в production-бандле, у которой
  снизу срезана кнопка «Выбрать».
