Контекст продукта:
Мы делаем сервис аренды техники и вещей через постаматы. Пользователь может взять на время приставку, проектор, инструмент, пылесос, бытовую технику или другой товар. Логика:
1. Пользователь выбирает город.
2. Смотрит каталог товаров.
3. Открывает страницу товара.
4. Видит описание, комплектацию, инструкцию, цены по длительности аренды.
5. Выбирает постамат на карте или из списка.
6. Выбирает длительность аренды.
7. Выбирает дату и время начала.
8. Видит итоговую стоимость.
9. Авторизуется или переходит к оформлению заказа.
10. После оплаты backend должен выдать/подготовить код получения через постамат.

Сделай frontend как для реального production-проекта.

Технологический стек:
- Используй текущий стек проекта, если он уже есть.
- Если стек не задан, используй React + TypeScript + Vite.
- Стили: Tailwind CSS или существующая дизайн-система проекта.
- Состояние: React Query/TanStack Query для API-запросов, Zustand или обычный React state для UI-состояния.
- Роутинг: React Router.
- Формы: react-hook-form + zod, если нужны формы.
- Карта: подготовь компонент-обертку под Яндекс.Карты/Leaflet/другой провайдер. Если ключей нет — сделай graceful fallback: список постаматов + заглушка карты.
- Код должен быть чистым, компонентным, расширяемым.
